# 面试猿刷题平台 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
**面试猿刷题平台**是一个专业的面试八股文智能刷题系统，致力于为求职者提供系统化、结构化的面试题目练习体验，帮助用户高效准备技术面试。

### 1.2 目标用户
- **主要用户**：准备技术面试的求职者（校招/社招）
- **次要用户**：技术面试官、培训机构、HR
- **用户痛点**：面试题目散乱、缺乏系统整理、答案质量参差不齐

### 1.3 产品价值
- **对于用户**：提供高质量、系统化的面试题库，支持分模块练习和答案解析
- **对于平台**：构建面试题库生态，积累优质内容，形成学习社区

## 2. 功能架构

### 2.1 用户端功能
#### 2.1.1 首页模块 (`/`)
- **功能**：展示所有题库列表
- **交互**：
  - 卡片式布局展示题库
  - 显示题库标题、描述、题目数量、标签
  - 点击进入对应题库详情
- **设计特点**：
  - 简洁的卡片设计
  - 支持响应式布局
  - 清晰的信息层级

#### 2.1.2 题库详情模块 (`/banks/[bankId]`)
- **功能**：展示特定题库下的所有题目
- **交互**：
  - 题目列表展示（标题、难度、标签）
  - 筛选和搜索功能
  - 点击题目进入详情
- **状态管理**：
  - 题目状态：草稿、待审核、已发布、已拒绝
  - 难度分级：简单、中等、困难

#### 2.1.3 题目详情模块 (`/question/[questionId]`)
- **功能**：展示题目完整信息和答案解析
- **交互特性**：
  - ✅ **答案默认隐藏**：用户需要主动点击查看答案
  - 🔒 **隐藏状态设计**：显示锁定图标、"答案已隐藏"提示、"查看答案"按钮
  - 📖 **答案展示**：Markdown格式渲染，支持富文本
- **内容结构**：
  - 题目基本信息：标题、描述、标签
  - 统计信息：浏览量、收藏数
  - 答案解析区域（可折叠/展开）

### 2.2 管理端功能 (`/admin`)
#### 2.2.1 题库管理 (`/admin/banks`)
- **核心功能**：题库的增删改查操作
- **交互特点**：
  - 表格形式展示所有题库
  - 支持搜索和分页
  - 状态管理和审核流程
  - 封面图片上传功能

#### 2.2.2 题目管理 (`/admin/questions`)
- **核心功能**：题目的批量管理和组织
- **交互特点**：
  - 题目列表展示（不显示答案内容）
  - 批量选择和操作
  - **新增**：批量导入功能按钮（黑底白字样式）
  - 题目与题库的绑定关系管理
- **操作流程**：
  1. 选择题目 → 点击"添加到题库"
  2. 输入题库ID → 确认绑定
  3. 支持批量导入（待实现）

## 3. 用户体验设计

### 3.1 交互原则
#### 3.1.1 学习体验优先
- **答案隐藏机制**：模仿真实面试场景，用户需要先思考再看答案
- **渐进式披露**：避免一次性展示过多信息，保持专注

#### 3.1.2 内容组织逻辑
```
题库分类 → 题目列表 → 题目详情 → 答案解析
   ↓          ↓         ↓        ↓
  分类学习   选择练习   深度思考   对照学习
```

### 3.2 视觉设计系统
#### 3.2.1 颜色规范
- **主色**：`#111827` (深灰色) - 传达专业感
- **强调色**：`#facc15` (黄色) - 突出重要信息
- **状态色**：
  - 简单：翠绿色 (`#10b981`)
  - 中等：蓝色 (`#3b82f6`)
  - 困难：紫色 (`#8b5cf6`)

#### 3.2.2 组件设计
- **卡片化布局**：统一的内容容器设计
- **按钮样式**：
  - 主要操作：黑底白字（批量导入）
  - 次要操作：轮廓样式（新建题库）
  - 辅助操作：文字链接

### 3.3 响应式设计
- **移动端优先**：`< 900px` 优化触控体验
- **桌面端增强**：`≥ 900px` 支持多列布局
- **关键断点**：
  - 320px：单列布局
  - 768px：双列布局
  - 1200px：多列布局

## 5. 用户旅程分析

### 5.1 求职者用户旅程
```
发现平台 → 浏览题库 → 选择练习 → 查看题目 → 思考答案 → 查看解析 → 标记收藏 → 继续练习
```

**关键触点**：
- 首页题库展示吸引用户
- 答案隐藏机制保持用户专注
- 解析质量直接影响用户留存

### 5.2 管理员用户旅程
```
登录管理后台 → 选择管理模块 → 浏览/搜索内容 → 执行管理操作 → 审核发布
```

**效率关注点**：
- 批量操作提升管理效率
- 清晰的权限控制
- 直观的数据展示
